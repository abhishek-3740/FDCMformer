================================================================================
0. THE CRITICAL PROBLEM (why this work was needed)
================================================================================

  RAM BLOW-UP IN THE DATA LOADER.
  The old data/npz_dataset.py loaded every shard fully into RAM (np.load +
  np.concatenate). Measured impact: 2 local shards alone took the process from
  ~210 MB -> ~1475 MB. The full train set (part1 ~60 + part2 ~60 shards at
  ~700 MB each) would need tens of GB of RAM and OOM instantly on Kaggle
  (~30 GB). The earlier smoke test only survived because it touched 2 shards.

  This had to be fixed before ANY GPU run could start. It is the headline fix
  of this change set; the rest is the multi-GPU / schedule / resume plumbing
  that a real Kaggle run also needs.


================================================================================
1. WHAT WAS CHANGED (file by file)
================================================================================

REWRITTEN  data/npz_dataset.py        (+249 / -... , the big one)
  - Lazy memory-mapped loader. An uncompressed .npz (np.savez, NOT
    savez_compressed) is just a ZIP of contiguous .npy entries, so we np.memmap
    straight into the parent file at each array's data offset. __init__ now
    reads ONLY the tiny .npy headers; each sample is paged off disk on demand.
        * _read_npy_header()        - parses a .npy header (self-contained, no
                                      private numpy APIs -> version robust).
        * _zip_entry_data_offset()  - computes the absolute byte offset of an
                                      entry's raw data via its LOCAL file header.
        * _ShardMeta                - per-shard lazy metadata (dtype/shape/offset)
                                      with no array data loaded.
        * NPZ_Dataset               - cumulative-length index + bisect to map a
                                      global idx -> (shard, local idx); memmaps
                                      are opened lazily per worker (fork/spawn
                                      safe, OS page cache shared across workers).
    Result: RAM stays flat (210 -> 210 MB) and values match np.load exactly,
    including across shard boundaries.
  - Multi-root + flat-shard support. _resolve_shards() searches, per root:
        1. <root>/<split>.npz             (single packed file)
        2. <root>/<split>/shard_*.npz     (part1 layout: train/ val/ test/)
        3. <root>/shard_*.npz             (flat shard dir, e.g. Kaggle part2)
    Passing a LIST of roots merges all discovered shards (deduplicated by
    resolved path). This is how the spring set, split across two Kaggle
    datasets, is joined: train reads part1/train + part2, val/test read part1.
  - Compressed-shard fallback: if a shard is ZIP_DEFLATED it can't be memmapped,
    so it falls back to a cached np.load for that shard only.

MODIFIED  util/data_split.py          (+22)
  - get_train_roots(config): appends config.dataset.train_extra_roots (part2)
    to the primary root for the TRAIN split ONLY; val/test keep the primary
    root. build_train_valid_datasets() now passes the merged train roots to the
    train dataset and the single root to the valid dataset.

MODIFIED  config/base_config.py       (+14)
  - dataset.train_extra_roots = []    (extra roots searched for train only)
  - train.gpu        = [0]            (int count for multi-GPU, or list of idx)
  - train.strategy   = 'auto'         ('auto' | 'ddp' | 'ddp_notebook')
  - train.lr_milestones = [120,150,170,180,190,200] ; train.lr_gamma = 0.5
                                      (LR schedule is now CONFIG-DRIVEN, see fix
                                       #2 below - previously hardcoded)
  - train.save_top_k = 1 ; train.ckpt_every_n_min = 0 ; train.max_time = None
                                      (checkpoint / graceful-stop knobs, off by
                                       default to preserve original behaviour)

NEW  config/ecrformer_kaggle_config.py   (+98)
  - ECRformerKaggleConfig(ECRformerConfig) - the full M9+M7+M1 model wired for
    Kaggle 2x T4:
        * _find_dataset_roots() auto-locates part1 (has train/val/test subdirs
          with shards) and part2 (flat shard dir) under /kaggle/input at ANY

        * precision = '16-mixed'   (T4 has NO bf16 -> fp16 mixed precision).
        * strategy  = 'ddp', gpu = 2  (both T4s; subprocess `!python train.py`).
        * train_bs=4, valid_bs=8, accumulate_grad_batches=2  ->
          effective batch = 4 x 2 x 2 = 16  (matches the single-GPU 4x4 recipe).
        * num_workers = 2          (Kaggle ~4 CPU cores, per DDP process).

        * LearningRateMonitor added; log_every_n_steps now config-driven.
  - max_time wired into the trainer for a clean, checkpointed stop before the
    9 h hard kill.
  - dump_config_snapshot(): writes a human-readable config_snapshot.json next to
    the TensorBoard logs each run (rank-0 only, never breaks training).
  - save_hyperparameters(ignore=['config']) - the resume-breaking fix (#3).

================================================================================
2. BUGS / CORRECTNESS FIXES (the "whatever fixes we applied" part)
================================================================================

  FIX #1 - Data loader RAM blow-up  (see section 0)
    Cause   : np.load + np.concatenate loaded entire shards into RAM.
    Fix     : lazy memory-mapping of uncompressed .npz entries.
    Verified: RAM flat at ~210 MB; sample values byte-identical to np.load,
              including across shard boundaries.

  FIX #2 - LR scheduler never decayed on short runs
    Cause   : milestones were HARDCODED [120,150,170,180,190,200] for a
              200-epoch run. At max_epoch=80 the LR would NEVER decay.
    Fix     : schedule is now config-driven (lr_milestones, lr_gamma); the
              Kaggle config sets [40,60,70,75] for its 80-epoch budget.

  FIX #3 - Resume broke under torch >= 2.6
    Cause   : the LightningModule pickled the whole config object into the
              checkpoint; torch>=2.6 loads with weights_only=True, which then
              fails to unpickle it -> resume crash.
    Fix     : save_hyperparameters(ignore=['config']). The model is always
              rebuilt from config before fit(), so nothing is lost; a readable
              config_snapshot.json is written alongside the logs instead.
    Verified: a full save -> resume cycle restores all states in place.

  FIX #4 - fp16 (mixed precision) NaN/inf in the M9 aux losses
    Two parts, both "always compute in fp32":
      (a) SAMLoss: acos'(+-1) = -1/sqrt(1-cos^2) -> -inf. Under fp16 the old
          clamp_eps=1e-6 was BELOW fp16 resolution near 1.0 (~1e-3), so
          1 - clamp_eps rounded back to exactly 1.0 and the clamp became a
          no-op. Fix: clamp_eps raised to 1e-4 (caps |grad| ~= 70), eps to 1e-6,
          and pred/target cast to .float() inside forward.
      (b) FFTLoss: complex amplitudes can exceed the fp16 max (65504) -> inf,
          and torch.abs() of a complex tensor has a 1/|z| term in its backward
          that explodes at near-zero frequency bins. Fix: compute in fp32 and
          take the magnitude manually as sqrt(re^2 + im^2 + eps) so the gradient
          stays finite.
    Belt-and-braces: in train.py the aux-loss block also runs inside
    torch.autocast(..., enabled=False) on fp32-cast tensors, so SAM/FFT/
    perceptual terms are evaluated in fp32 even under '16-mixed'.

  FIX #5 - DataLoader worker safety with memmaps
    The memmap handles are created LAZILY per worker (not pickled from the
    parent), so the loader is fork/spawn safe and the OS page cache is shared
    across workers; persistent_workers is enabled only when num_workers > 0.

================================================================================
4. HOW TO TRAIN ON KAGGLE (2x T4)
================================================================================

  1. Push this code where Kaggle can read it: zip the FDCMformer folder and
     upload it as a Kaggle Dataset (e.g. fdcmformer-code). The notebook
     auto-detects it by locating train.py.
  2. New Notebook, right sidebar:
       - Accelerator -> GPU T4 x2
       - Internet    -> ON   (LPIPS/VGG weights download on first validation)
       - Add data    -> attach all three: the code dataset,
                         sen12ms-cr-spring-part1, sen12ms-cr-spring-part2.
  3. Import kaggle_train.ipynb (File -> Import Notebook), or paste its cells.
     Run cells 1-3 first to confirm shard counts (expect ~60 train in part1,
     4 val, 19 test, ~60 in part2).
  4. Launch training (subprocess, true DDP):
       !python train.py -c ecrformer_kaggle --devices 2 --strategy ddp
     In-kernel alternative:
       !python train.py -c ecrformer_kaggle --devices 2 --strategy ddp_notebook
  5. Save Version -> Save & Run All (Commit) to train headless on both T4s.

  Resume across the 9 h limit:
    - Training auto-resumes from last.ckpt (rolling, every epoch + every 30 min).
    - To continue past a timeout, attach the previous run's OUTPUT as an input
      and copy its experiments/ into /kaggle/working/ (the notebook automates
      this). max_time stops cleanly ~30 min before Kaggle's hard kill.

  Local / single-GPU (unchanged):
    python train.py --config ecrformer            # full M9+M7+M1 model
    python train.py --config ecrformer_light
    Flags: -n NAME (run suffix), -g GPU (index), --no-resume (clean start).

  TensorBoard:
    tensorboard --logdir experiments
    Scalars: train_MAE/SSIM, down_proj, up_proj, valid_* (RMSE/MAE/PSNR/SAM/
    SSIM/LPIPS), learning_rate, and the M9 aux_* terms (aux_sam/fft/perceptual).


================================================================================
5. TROUBLESHOOTING
================================================================================

  - NaNs under mixed precision: rerun with --precision 32. (The SAM/FFT fp32
    fixes in #4 should prevent this, but 32 is the safe fallback.)
  - GPU OOM: lower train.train_bs and raise optim.accumulate_grad_batches to
    keep the effective batch (=train_bs * accumulate * num_gpus) at 16.
  - "No shards found": check that part1 has train/ val/ test/ subfolders with
    shard_*.npz and part2 is a flat dir of shard_*.npz; cells 1-3 print counts.
  - Resume not picking up: confirm the previous run's experiments/ was copied
    into /kaggle/working/ and that the log_name (config name + -n suffix) matches.
