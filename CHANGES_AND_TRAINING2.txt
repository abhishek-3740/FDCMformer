================================================================================
NEXT-CR : ECRformer + M9 + M7 + M1  (this file documents M1 only)
Bi-Cross-Modal Attention (BCMA) fusion stem.
Changes, rationale, training instructions, and a final review.
================================================================================

This is a follow-up to CHANGES_AND_TRAINING.txt (which covered M9 + M7).
It documents ONE new roadmap modification layered on top of that work:

  * M1 - Bi-Cross-Modal Attention (BCMA) stem, replacing the channel-concat
         fusion of SAR and optical at the network entry.

Everything stays configurable and backward-compatible: set bcma=False to
recover the previous (M9 + M7) behaviour, and additionally set
sam=fft=perc=0 + gated_skip=False to recover the original ECRformer.


================================================================================
0. WHY M1  (the bottleneck it fixes)
================================================================================

  B2 - Trivial multimodal fusion. In the baseline, SAR (2 ch) and optical
  (13 ch) are each passed through a SINGLE 7x7 conv and then concatenated
  (DecoupledEncoder). From the stem onward they are one tensor; the two
  modalities only ever interact implicitly through the channel-covariance
  attention (MDTA) inside the blocks. There is no explicit, spatial mechanism
  that routes SAR structure into cloud-occluded optical regions - which is
  exactly the regime (thick cloud) where SAR matters most.

  SAR and optical carry ASYMMETRIC information:
    - SAR     ~ geometry / structure, available under any weather.
    - optical ~ spectral / colour, destroyed under cloud.
  Channel-concat throws this asymmetry away. M1 replaces it with a principled,
  symmetric cross-modal attention plus a learned modality gate.


================================================================================
1. WHAT WAS CHANGED (file by file)
================================================================================

MODIFIED  models/ecrformer_model.py
  - Added CrossXCA module (cross-covariance CROSS-attention):
        Q  = proj(x_q)            # from one modality
        K,V= proj(x_kv)           # from the OTHER modality
        attn over CHANNELS (C x C), softmax, out = attn @ V
    Mirrors the model's existing TransposedAttention/MDTA, so attention is
    over channels and cost is LINEAR in spatial tokens (the cheap, XCA-style
    attention the M1 spec calls for). A depthwise 3x3 conv on q/k/v injects
    local spatial context before the channel attention.

  - Added BCMAStem module (drop-in replacement for DecoupledEncoder):
        Fs = sar_stem(SAR)        # 7x7 conv + learnable depthwise 5x5
                                  #   SPECKLE-REDUCTION conv (spec mitigation
                                  #   so SAR speckle does not get gated off)
        Fo = opt_stem(optical)    # 7x7 conv
        Fo_c = CrossXCA(Fo <- Fs) # optical queries SAR
        Fs_c = CrossXCA(Fs <- Fo) # SAR queries optical
        go = sigmoid(Conv1x1([Fo;Fs]) + gate_bias_o)   # learned modality gate
        gs = sigmoid(Conv1x1([Fs;Fo]) + gate_bias_s)
        Fo'= go*Fo + (1-go)*Fo_c
        Fs'= gs*Fs + (1-gs)*Fs_c
        out= concat([Fo', Fs'])   # channel count == features_start
    Output channels equal features_start, so the ENTIRE downstream encoder/
    decoder is unchanged -> true drop-in, just like GatedSkipFusion was for M7.

  - Added constructor arg `bcma: bool = False` and `self.bcma`. When True (and
    exactly two inputs [SAR, optical]) the stem is BCMAStem; otherwise the
    previous DecoupledEncoder / plain conv stem is used unchanged.

  INIT DETAIL (important): util.util.initialize_weights ZEROES every Conv2d
  bias after construction, which would wipe a gate-bias baked into the gate
  conv. So the "start near the original decoupled stem" behaviour is anchored
  by a STANDALONE nn.Parameter (gate_bias_o / gate_bias_s, init +3.0) added to
  the gate logits. Kaiming init does not touch raw Parameters, so the gate
  starts at sigmoid(3) ~= 0.95 (mostly self-features) and LEARNS how much
  cross-modal signal to admit from there.

MODIFIED  config/ecrformer_config.py
  - Added bcma=True to net.cfg (inherited by ecrformer_light_config.py), in the
    same style as gated_skip=True. This makes the default `ecrformer` config the
    full M9 + M7 + M1 model (experiment E5).

  (No change needed to train.py, base_config.py, or the data pipeline: BCMA is
   purely internal to the network's stem and the stem's output contract -
   features_start channels - is identical to before.)


================================================================================
2. SCOPE OF THIS IMPLEMENTATION  (read before you train)
================================================================================

  BCMA is currently applied at the STEM ONLY, not "at every encoder stage" as
  the original M1 spec described. This is a deliberate, cost-conscious first
  cut:

    - The two modalities now interact through principled cross-attention
      BEFORE merging (fixes the "shallow concat" complaint at the entry point).
    - But the merge still happens at the stem, so deeper encoder/decoder layers
      see a single fused tensor.

  Consequence: the parameter cost is tiny - about +9 K params
  (11.4709 M -> 11.4802 M for the full config), far below the spec's "+0.6 M".
  That estimate assumed BCMA at every (wider, deeper) encoder stage.

  If E5 shows BCMA helps but under-delivers vs the +0.4..0.7 dB target, the
  natural next step is to keep DUAL streams through the first 1-2 encoder stages
  and apply CrossXCA there too. That is where the spec's larger param/FLOP
  budget and bigger expected gain live. We chose to prove the stem version
  first rather than commit to the heavier dual-stream encoder blind.

================================================================================
8. FINAL REVIEW - what is solid, what to watch, what is deferred
================================================================================

  WHAT IS SOLID
  - True drop-in: BCMAStem preserves the stem output contract (features_start
    channels), so nothing downstream changed. Same pattern that made M7 safe.
  - Backward-compatible: bcma=False restores the exact prior model; the E0
    baseline is still reproducible.
  - Cheap and linear: CrossXCA is channel attention (C x C), linear in tokens.
  - Speckle mitigation is built in (learnable depthwise conv in the SAR stem),
    addressing the spec's main risk (gate suppressing noisy SAR).
  - Sensible init (gate ~0.95 self) means training does not start from a random
    fusion - it eases into cross-modal mixing.

  THINGS TO BE AWARE OF (design choices / caveats, not bugs)
  1. STEM-ONLY scope (see section 2). If gains are small, extend BCMA into the
     first encoder stage(s) - that is the spec's full-scope version.
  2. Equal-width branches. CrossXCA's channel attention requires the SAR and
     optical branches to have equal channel counts; the stem uses
     split_integer(features_start, 2, [1,1]) = features_start/2 each. This is
     even for both configs (48->24, 32->16). If you ever set an odd
     features_start, add a guard.
  3. M1 adds NO new loss term and NO new TensorBoard scalar - it is purely
     architectural. Judge it by the valid_* metrics (PSNR/SAM/SSIM), not by a
     new aux_* curve.
  4. Gate behaviour is worth inspecting post-training: log/visualise go, gs to
     confirm the model learns to lean on SAR under thick cloud (sanity that the
     mechanism is doing what we claim - good for the paper).

  WHAT IS DEFERRED (explicitly NOT in this change set)
  - M2 (Wavelet-Decoupled Encoder-Decoder, fixes B3 + B4) is the agreed next
    contribution, to be implemented AFTER E5 confirms M1 helps.
  - Full-scope per-stage BCMA (dual streams through early encoder stages).

  RECOMMENDED NEXT STEP
  - Run E4 vs E5 on a SEN12MS-CR subset for a few epochs first (confirm loss
    curves behave, no NaNs, gates moving), then a full E5 run. Compare to E4 on
    PSNR/SAM/SSIM, paying attention to thick-cloud samples. Only then decide
    between (a) shipping stem-only M1, (b) extending to per-stage BCMA, or
    (c) moving on to M2.
